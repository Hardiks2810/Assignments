1) Are the HTML tags and elements the same thing?

HTML Tags: HTML tags are used to define the structure and elements within a web page. Tags are composed of angle brackets (< and>) and usually come in pairs: an opening tag and a closing tag. Tags are used to enclose content and provide instructions to the web browser on how to display that content.

HTML Elements: HTML elements are composed of tags and the content they enclose. An HTML element consists of an opening tag, the content (which may include text, other elements, or multimedia), and a closing tag. Elements are the fundamental building blocks of an HTML document.


2) What are tags and attributes in HTML?

In HTML, tags and attributes are essential components that are used to define and structure web content. They play different roles in specifying how content is presented and behaves on a web page:

HTML Tags:
- Tags are the fundamental building blocks of HTML. They are used to define and structure content on a web page.
- Tags are enclosed in angle brackets (< and>) and usually come in pairs: an opening tag and a closing tag.
- HTML tags are used to create various elements, such as paragraphs, headings, lists, links, images, forms, and more. Each tag is associated with a specific HTML element.
HTML Attributes:
- Attributes are used to provide additional information about HTML elements within their opening tags.
- Attributes are always specified in the opening tag and consist of a name and a value, separated by an equal sign. The attribute name is followed by the equal sign and the attribute value enclosed in double or single quotes.
- Attributes serve various purposes, such as defining the source of an image, specifying the link destination for an anchor (link) element, setting styles, and providing data for form elements.

3) What are void elements in HTML?

Void elements, also known as self-closing elements or empty elements, are a category of HTML elements that do not have a closing tag. Instead of having both an opening and closing tag, void elements are written as a single self-closing tag, typically ending with a forward slash before the closing angle bracket. Void elements are commonly used for including things like images, line breaks, input fields, and other elements that don't need additional content.

Some common examples of void elements in HTML include:
  1. < img> Used for embedding images.
  2. < br> Used to insert line breaks.
  3. < input> Used for creating form input fields.
  4. < hr> Used to create horizontal rules or thematic breaks.
  5. < link> Used for linking external resources, such as stylesheets.

Void elements are a specific subset of HTML elements that have a predefined list of attributes and don't require a separate closing tag because they can't contain any nested content. They are designed for simplicity and efficiency in web development.


4) What are HTML Entities?

An HTML entity is used to display invisible characters and reserved characters that would otherwise be interpreted as HTML code. It is a piece of text, or string, that begins with an ampersand ( & ) and ends with a semicolon ( ; ).

11) What is the difference between a block-level element and an inline element?

Block elements always start from a new line. Inline elements never start from a new line. Block elements cover space from left to right as far as it can go. Inline elements only cover the space as bounded by the tags in the HTML element.

16) How are active links different from normal links?

An active link is a link that can be clicked on to take you to another page or website. A passive link is a link that can't be clicked on.

18) What is difference between HTML and XHTML?

HTML stands for Hypertext Markup Language, whereas XHTML stands for Extensible Hypertext Markup Language. XHTML is basically an extension of HTML, which is stricter than HTML. Both languages are used to create web and Android applications. HTML is SGML based, whereas XHTML is an XML-based language.
